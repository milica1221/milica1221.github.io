Nova Vector Platform

Nova Vector platform has the potential to capture data that has eluded manufacturers for many years. The results are greater revenue earned from higher levels of compliance, manufacturing quality, and delighted customers.

Open Source

In addition to being licensed under Apache License 2.0, Nova Vector is also an open-source platform. This means that users have access to the source code, and can modify and distribute it freely. The open-source nature of the platform encourages collaboration and innovation, allowing users to build upon existing features or create new ones to meet their specific needs. Additionally, open-source software is generally more secure, as it is subject to constant scrutiny and updates from a community of developers.

Multi-Tenancy

Nova Vector is designed as a multi-tenant architecture that physically integrates tenants while keeping them logically separated. It runs a single instance of the software on one server or server farm and serves multiple tenants. With the use of roles, clients can set restricted access to specific assets for fine-grained management.

Support for OAuth 2/ODIC

Nova Vector supports OAuth 2.0, the industry-standard protocol for authorization, which enables single sign-on functionality for clients. Users don't have to deal with login forms, authenticating users, and storing users. They can log in with social networks, and once logged in, they don't have to log in again to access a different application.

Integrated LoRaWAN Server

Nova Vector Industry 4.0 Platform has integrated LoRaWAN server. It enables long-range, low-power wireless solutions that can be deployed as public or private networks. LoRa enabled devices, depending on configuration, can have battery life of up to 10 years. This enables quick deployment of LoRa IoT sensors with no need for wires.

Built In Rules Engine

Nova Vector provides integrated Rules Engine, which is a framework for building event-based flows, that can contain messages, Rule Nodes and Rule Chains.It can process both simple and complex events. Rule Engine can help you filter and alter the incoming messages from the devices as well as connect with external systems.

Function as a Service Module

Nova Vector's FaaS module uses serverless architecture to run small, modular functions. This approach offers several benefits, including simplified developer logistics, increased agility, faster updates, and greater responsiveness to customer needs. Additionally, the FaaS model allows for individual functions to scale automatically and independently based on their usage, making it highly scalable.

Edge Computing Component

Nova Vector has an Edge Computing component, in which client data is processed at the periphery of the network, as close to the originating source as possible. Rather than transmitting raw data to a central data center for processing and analysis, that work is instead performed where the data is actually generated -- whether that's a retail store, a factory floor, a sprawling utility or across a smart city.
